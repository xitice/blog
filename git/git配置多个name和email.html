# 利用git config 配置全局、目录、仓库及的用户名和邮箱

# 为什么要配置不同的用户名和邮箱？

  • 一般资料、环境最齐全的就一台电脑
  • 我们除了要提交公司的代码，还要往自己的 git 平台提交代码
  • 两个代码管理平台需要用不同的邮箱，是保护公司，也是保护自己

# 全局配置

git config --global user.name "your username"
git config --global user.email "yout email"

这种配置方法，在当前计算机生效，在 user/currentUser 目录下生成.gitconfig文件 但是这种配置会使你在任何仓库提交都是一个 name 和 email

# 仓库及配置

git config user.name "your username"
git config user.email "your email"

以上这种配置，在当前 git repository 生效，会在当前 repository/.git/config 中生成对应的记录，只在当前目录有效

# 利用 include/includeif 实现不同文件夹下不同的git配置

TIP

只有git版本大于2.13才能用

利用include/includeif可以实现在不同的目录加载不同的局部配置文件

步骤：

下面为.gitconfig文件：

[user]
	name = yourGlobalName
	email = yourGlobalEmail
[includeIf "gitdir:E:/study/gitdoc/"]
	path = .gitconfig-study.config

下面为path路径中指定的.gitconfig-study.config

[user]
	name = yourPrivateName
	email = yourPrivateEmail

注意：在博客中看到很多人在写只要 .gitconfig-study.config 放置的目录保持和path一致就行，但在本人电脑上莫名就是不行，最后无奈将 .gitconfig-study.config 放置在 .gitconfig 同一目录，最后生效。怀疑是在 windows 电脑上path 的写法有一定的特殊性，所以用相对目录可以成功。

# 验证

进入 gitdir 对应的目录，打开 git bash，输入 git config user.name 查看是否是自己配置的name

# git 查找 user.name 和 user.email 的规则

先判断是否是在git仓库？

  • 是在 git 仓库
    • 有局部 config 的话，加载局部配置的 config
    • 没有局部的 config 的话，按照 include/includeif 加载
  • 如果不是在git仓库
    • 如果全局配置了include/includeif，按照 include/includeif 加载
    • 如果全局没有配置 include/includeif， 直接获取全局配置
